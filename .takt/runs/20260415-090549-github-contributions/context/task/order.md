# タスク指示書

## タスク
トップページの `GitHub contributions` 表示に、各マスが上下左右の四方から集まってきて 3 秒で完成形になるアニメーションを追加する。アニメーションは、そのセクションが画面内に入るたびに毎回再生されるようにする。

## 参照資料
- `src/components/GitContribution/index.tsx`
- `src/app/page.tsx`
- `src/style/global.css`
- `src/components/MotionSection.tsx`
- `src/lib/motion.ts`
- `src/components/MotionSection.test.tsx`

## 要件
- 対象は `GitHub contributions` の表示部分
- アニメーション対象は contributions 全体ではなく、各マス・各要素ごと
- 各マスは上下左右のいずれかの方向から集まってくる表現にする
- セクションが画面内に入るたびにアニメーションを再生する
- アニメーション全体は 3 秒で完成する
- アニメーション完了後は、現在の contributions グリッドと同じ完成形になる

## テストで確認すべきこと
- contributions セルごとにアニメーション用の属性・クラス・スタイルが付与されること
- セルごとに上下左右のいずれかの方向情報を持てること
- セクションが画面内に入ったときにアニメーション再生状態になること
- 一度画面外に出て再度入ったとき、再びアニメーションが再生されること
- 3 秒で完成する設定値が適用されていること

## 再現手順
1. `bun dev` を起動する
2. `/` を開く
3. `Contributions` セクションまでスクロールする
4. 各セルが四方から集まり、3 秒で完成形になることを確認する
5. セクションが画面外に出るまでスクロールし、再度セクションを画面内に入れる
6. アニメーションが毎回再生されることを確認する

## 確認方法
- `bun test`
- `bun run build`
- ブラウザで `/` を開いて手動確認
- 確認観点:
  - 各セルが個別に動く
  - 3 秒で完成する
  - セクション再入ごとに再生される
  - 最終的なグリッド配置と色が現行表示と一致する
